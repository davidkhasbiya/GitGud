# ADR-005: UI Philosophy

Status: Accepted

Date: 2026-07-03

---

## Context

Many learning platforms contain excessive visual elements that distract users from practicing.

---

## Decision

Adopt a "Low Noise, High Function" interface philosophy.

---

## Principles

- One Page, One Primary Action.
- Minimal visual distractions.
- Monochrome-first color palette.
- Developer-focused layout.
- Content over decoration.
- Responsive by default.

---

## Inspirations

- Linear
- GitHub
- Vercel
- Raycast
- Cursor IDE

---

## Consequences

Every new UI component must:

- Have a clear purpose.
- Reduce unnecessary visual complexity.
- Prioritize readability and usability.
